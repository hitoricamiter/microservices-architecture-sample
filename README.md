Техническое задание

Приложение реализовано с использованием микросервисной архитектуры. В процессе работы мной были протестированы и настроены следующие компоненты: Kafka, Eureka Server, API Gateway и Config Server. Планировалось внедрение Camunda для оркестрации бизнес-процессов, однако позиция была закрыта до реализации этой функциональности.

Обмен данными между микросервисами осуществляется через Kafka, при этом данные сериализуются и передаются по соответствующим эндпоинтам. Приложение запускается частично локально и частично через Docker: два микросервиса работают на локальной машине, а остальные компоненты (Kafka, Zookeeper и др.) — через Docker Compose.

Конфигурация Config Server была основана на шаблоне: https://github.com/hitoricamiter/config-server
